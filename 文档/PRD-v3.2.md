# 1. 产品概述

## 1.1 产品定位

AI 漫剧分镜脚本助手是一款面向漫剧创作小白的 AI 工具。用户只需输入一句简单文案，AI 自动完成故事拆解、分镜脚本生成、漫画图片生成的全流程。产品同时支持 iOS App 和网页版。

## 1.2 技术架构

| **层级** | **技术选择** | **说明** |
| --- | --- | --- |
| 前端 | React Native (Expo SDK 55) | 一套代码同时出 iOS App + 网页版 + 未来 Android |
| 后端 | Python FastAPI 0.3.0 | API 服务，REST + SSE |
| AI 编排 | LangGraph | 状态图编排多步骤 AI 工作流 |
| 文本 AI | 通义千问 qwen-plus | 故事拆解、脚本生成、单格重写 |
| 生图 AI | 通义万相 wanx2.1-t2i-turbo | 漫画图片生成（DashScope 异步模式） |
| 数据库 | SQLite（MVP）→ MySQL（生产） | SQLAlchemy ORM，用户表 + 作品表 |
| 图片存储 | 通义万相临时 URL（MVP）→ 阿里云 OSS（计划） | 生成的漫画图片 |
| 本地存储 | AsyncStorage | 作品数据、登录态、用户名本地持久化 |
| 认证 | JWT (python-jose) | 手机号+验证码登录，30天有效期 |
| 部署 | 阿里云 ECS | 2核4G，MVP 阶段 |
| 开发工具 | Claude Code + Pencil MCP | 代码生成 + 设计稿读取 |

## 3.4 引导式创作流程（3 步）

用户点击"生成分镜脚本"后，自动携带首页输入的文案进入 Wizard 页面。顶部导航栏显示"步骤 N/3 — 标题"，下方 3 个圆点进度指示器。

### 步骤 1/3 — 选择时长

- 胶囊按钮：10秒 / 15秒 / 30秒 / 60秒
- 默认值：15秒
- 说明文字："默认 15 秒，适合标准漫剧节奏"

### 步骤 2/3 — 选择风格

- 卡片式网格选择（4列2行），每张卡片带 emoji + 风格名称
- 选中态：紫色背景 + 白色文字 + 右上角勾选标记
- 8 种风格：悬疑反转 / 搞笑沙雕 / 治愈温情 / 暗黑爆爽 / 古风仙侠 / 超能力奇幻 / 恐怖惊悚 / 甜宠恋爱
- 默认值：悬疑反转

### 步骤 3/3 — 选择镜头数 + 图片比例

- 镜头数胶囊按钮：4 / 6 / 9 / 12，默认 6

# 4. 用户操作流程

## 4.1 核心流程：创作漫画

```
打开 App
  → 登录页（手机号+验证码，或演示模式直接进入）
  → 进入创作首页
  → 在输入框写故事（如"渣男被前女友报复"）
  → 点击「生成分镜脚本」按钮（积分检查：5 积分）
  → 进入引导式创作页面（3 步）
    → 步骤 1/3: 选择时长（默认15秒）→ 点「下一步」
    → 步骤 2/3: 选择风格（如悬疑反转）→ 点「下一步」
    → 步骤 3/3: 选择镜头数（默认6）+ 图片比例（默认3:4）→ 点「开始生成」
  → 创建后台任务，进入加载页（4步进度动画）
    → 可选择「返回首页（后台生成）」→ 回首页，首页顶部出现紫色进度气泡
    → 可选择「取消生成」→ 任务标记为失败，返回首页
    → 等待完成后自动跳转结果页，作品自动存入 works 列表
  → 进入「脚本结果页」
    → 上下滑动查看每格分镜
    → 可编辑文字（场景、角色、对白、情绪）
    → 可点「AI 重写」按钮，AI 重新生成单格脚本
    → 点击「一键生成全部图片」（积分检查：10 积分）
    → SSE 流式推送，逐张显示生成的图片 + 进度
    → 可点单格「重新生成」重新生成某格图片
    → 全部图片完成后：
      → 「确定」→ 标记为已完成，保存到"我的作品"
      → 「导出」→ Web 下载 / Native 分享
      → 「重新生成全部图片」→ 重新走 SSE 生图流程
```

## 4.2 探索灵感流程

```
创作页「探索灵感」区域
  → 左右滑动浏览灵感卡片（Unsplash 真实图片）

    → 左右滑动查看分镜（mock 完整脚本）

  → 搜索框输入关键词 → 实时过滤（按标题、作者、标签）
  → 点击风格标签 → 过滤对应风格的作品
  → 点击作品卡片 → 进入作品详情页

## 4.4 个人中心流程

```
首页点击头像 → 进入我的页面
  → 查看积分余额，点击签到 +5 积分
  → 点击充值 → 打开积分商店弹窗，选择套餐购买

# 6. 脚本生成规范

## 6.1 叙事黄金公式

所有脚本必须严格遵循 4 段式叙事结构：

| **阶段** | **名称** | **占比** | **作用** |
| --- | --- | --- | --- |
| 第 1 格 | Hook | 前 20% | 第一秒就抓住注意力，禁止铺垫 |
| 中间 | Escalate | 中间 50% | 快速推进，每格有新信息，情绪递增 |
| 倒数第 2 格 | Twist | 后 20% | 出人意料的反转，打破观众预期 |
| 最后 1 格 | Cliffhanger | 最后 10% | 留钩子，让观众想看下一集 |

## 6.2 禁止项

- 禁止第 1 格用于介绍背景或人物关系
- 禁止连续 2 格情绪强度相同
- 禁止最后一格给完整结局（必须留悬念）
- 禁止任何一格不含对白或旁白
- 禁止连续 2 格相同镜头角度

## 6.3 镜头数与叙事结构映射

| **镜头数** | **Hook (20%)** | **Escalate (50%)** | **Twist (20%)** | **Cliffhanger (10%)** |
| --- | --- | --- | --- | --- |
| 4 | 第 1 格 | 第 2 格 | 第 3 格 | 第 4 格 |
| 6 | 第 1 格 | 第 2-4 格 | 第 5 格 | 第 6 格 |
| 9 | 第 1-2 格 | 第 3-6 格 | 第 7-8 格 | 第 9 格 |
| 12 | 第 1-2 格 | 第 3-9 格 | 第 10-11 格 | 第 12 格 |

代码实现：`graph.py` 中 `NARRATIVE_MAP` 字典 + `_get_narrative_phase()` 函数。

## 6.4 8 种脚本风格

| **编号** | **风格** | **英文标识** | **Emoji** | **核心特点** |
| --- | --- | --- | --- | --- |
| 1 | 悬疑反转 | suspense_twist | 🔍 | 每格埋伏笔，结尾大反转 |
| 2 | 搞笑沙雕 | comedy_absurd | 😂 | 夸张表情，无厘头式反转 |
| 3 | 治愈温情 | healing_warm | 🌸 | 温柔细腻，情感反转 |
| 4 | 暗黑爆爽 | dark_revenge | 🔥 | 快节奏强冲突，复仇爽感 |
| 5 | 古风仙侠 | wuxia_fantasy | ⚔️ | 古装场景，江湖气息 |
| 6 | 超能力奇幻 | superpower_fantasy | ✨ | 科幻/魔法元素，觉醒成长 |
| 7 | 恐怖惊悚 | horror_thriller | 👻 | 压抑氛围，惊吓结尾 |

代码定义：`types.ts` 中 `STYLE_OPTIONS` 数组 + `models.py` 中 `StyleEnum` 枚举和 `STYLE_CN` 映射。

## 6.5 时长选项

| **时长** | **说明** |
| --- | --- |
| 10秒 | 超短篇，快节奏冲击 |
| 15秒（默认） | 标准漫剧节奏 |

# 7. 后端服务设计

## 7.1 LangGraph 状态图

### 链路 1：生成脚本（POST /api/generate-script）

parse_story → refine_script → 返回完整分镜脚本

**parse_story：** 接收文案 + 风格 + 时长 + 镜头数，调用通义千问（ChatOpenAI），按黄金公式拆解分镜脚本。输出 JSON 格式，包含 panels 数组。无 API Key 时回退到 mock 数据（6 个模板循环复用）。

**refine_script：** 润色每格描述，补充视觉细节（光影、色调、构图），为每格生成英文 image_prompt（适合 text-to-image 模型）。无 API Key 时跳过。

### 链路 2：生成图片（POST /api/generate-images，SSE）

不经过 LangGraph，直接在 API 层循环处理：

对每一格 → 构造 prompt → 调用通义万相（DashScope 异步模式：提交任务 → 轮询状态，最多 120 秒）→ SSE 推送 `{panel_number, image_url, status}` → 全部完成后推送 `{event: "done"}`

无 API Key 时返回 mock placeholder 图片 URL。

### 链路 3：单格操作（直接调用，不经过 LangGraph）

**rewrite-panel（已实现）：** 调用通义千问重写单格脚本，保持 panel_number 和 narrative_phase 不变，重写场景/角色/对白/镜头/情绪，同时更新 image_prompt。无 API Key 时返回简单 mock（前缀"（AI 重写）"）。

**regenerate-image（已实现）：** 调用通义万相重新生成单格图片。

**generate-single-image（已实现）：** 为单格首次生成图片。
