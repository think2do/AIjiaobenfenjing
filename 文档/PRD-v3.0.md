# 1. 产品概述

## 1.1 产品定位

AI 漫剧分镜脚本助手是一款面向漫剧创作小白的 AI 工具。用户只需输入一句简单文案，AI 自动完成故事拆解、分镜脚本生成、漫画图片生成的全流程。产品同时支持 iOS App 和网页版。

## 1.2 技术架构

| **层级** | **技术选择** | **说明** |
| --- | --- | --- |
| 前端 | React Native | 一套代码同时出 iOS App + 网页版 + 未来 Android |
| 后端 | Python FastAPI | API 服务，REST + SSE |
| AI 编排 | LangGraph | 状态图编排多步骤 AI 工作流 |
| 文本 AI | 通义千问 qwen-plus | 故事拆解、脚本生成 |
| 生图 AI | 通义万相 wan2.5-t2i + Nano Banana | 漫画图片生成 |
| 数据库 | MySQL | 用户、作品、积分等 |
| 图片存储 | 阿里云 OSS | 生成的漫画图片 |
| 部署 | 阿里云 ECS | 2核4G，MVP 阶段 |
| 开发工具 | Claude Code + Pencil MCP | 代码生成 + 设计稿读取 |

## 3.4 引导式创作流程（4 步）

用户点击"生成分镜脚本"按钮后，从底部弹出引导式创作流程（向上推出动画）。顶部显示圆点进度指示器（●○○○），可回退上一步，每步都有默认值。

### 步骤 1/4 — 输入文案

- 大文本输入框，预填充从首页带入的文案
- 可继续编辑
- 底部按钮："下一步"

### 步骤 2/4 — 选择时长

- 胶囊按钮：10秒 / 15秒 / 30秒 / 60秒 / 自定义
- 默认值：15秒
- 自定义时弹出数字输入框

### 步骤 3/4 — 选择风格

- 卡片式选择，每张卡片带示例图 + 风格名称
- 8 种风格：悬疑反转 / 搞笑沙雕 / 治愈温情 / 暗黑爆爽 / 古风仙侠 / 超能力奇幻 / 恐怖惊悚 / 甜宠恋爱
- 默认值：悬疑反转
- 用户只能选 1 种

### 步骤 4/4 — 选择镜头数 + 图片比例

- 镜头数胶囊按钮：4 / 6 / 9 / 12，默认 6
- 图片比例选择：3:4 竖版 / 1:1 方形 / 16:9 横版 / 自定义，默认 3:4
- 底部按钮："开始生成"

## 3.5 脚本编辑页

- 纵向卡片列表，上下滑动
- 每格可编辑所有字段：场景、角色、对白、镜头角度、情绪
- 每格有"AI 重写"按钮
- 每格有"单格生图"按钮
- 底部固定按钮："一键生成全部图片"
- 编辑为纯前端操作，不调后端

## 3.6 生图过程

- 留在脚本编辑页，每格卡片上直接显示生成的图片

## 3.7 生图完成后

## 3.8 导出

# 4. 用户操作流程

## 4.1 核心流程：创作漫画

```
打开 App
  → 默认进入「创作」Tab
  → 看到问候语 + 探索灵感卡片 + 故事输入区
  → 在输入框写故事（如"渣男被前女友报复"）
  → 点击「生成分镜脚本」按钮
  → 弹出引导式创作流程（从底部推出）
    → 步骤 1: 确认/修改文案 → 点「下一步」
    → 步骤 2: 选择时长（默认15秒）→ 点「下一步」
    → 步骤 3: 选择风格（如悬疑反转）→ 点「下一步」
    → 步骤 4: 选择镜头数（默认6）+ 图片比例（默认3:4）→ 点「开始生成」
  → 等待 AI 生成脚本（显示加载动画）
  → 进入「脚本编辑页」
    → 上下滑动查看每格分镜
    → 可编辑任意格的文字内容
    → 可点单格「AI 重写」
    → 可点单格「单格生图」
    → 点底部「一键生成全部图片」
  → 图片逐张生成（SSE 实时显示）
  → 全部生成完成
    → 可逐格检查修改文字
    → 可点「重新生成」重做某格图片
    → 点「导出」→ 单张分开保存 + 生成分享链接
```

## 4.2 探索灵感流程

```
创作页「探索灵感」区域
  → 左右滑动浏览灵感卡片
  → 点击某张卡片 → 进入作品详情页（查看完整分镜）
  → 点击「查看全部」→ 跳转到「探索」Tab
```

## 4.3 探索发现流程

```
点击底部「探索」Tab
  → 进入探索页
  → 浏览瀑布流作品
  → 可使用搜索栏搜索关键词
  → 可点击风格标签筛选（如"悬疑反转"）
  → 点击某张作品卡片 → 进入作品详情页
    → 查看完整分镜漫画
    → 可点赞 ❤
    → 可收藏 🔖
    → 可评论（V3）
```

## 4.4 个人中心流程

```
点击底部「我的」Tab
  → 查看个人信息 + 数据统计
  → 点击个人信息卡片 → 编辑个人资料（头像、昵称）
  → 点击「我的作品」→ 查看已完成的作品列表
  → 点击「我的草稿」→ 查看未完成的草稿，可继续编辑
  → 点击「我的收藏」→ 查看收藏的他人作品
  → 点击「我的点赞」→ 查看点赞过的作品
  → 点击「使用记录」→ 查看历史生成记录
  → 点击右上角齿轮 → 进入设置页（账号管理、清除缓存、关于等）
```

# 5. 前后端技术交互流程

## 5.1 创作流程交互

```
┌─────────────┐                              ┌─────────────┐
│   前端 App   │                              │  后端 API   │
└──────┬──────┘                              └──────┬──────┘
       │                                            │
       │  1. 用户输入文案 + 选择参数                   │
       │  （纯前端，无请求）                           │
       │                                            │
       │  2. 点击「开始生成」                          │
       │──── POST /api/generate-script ────────────→│
       │     body: {                                │
       │       story: "渣男被前女友报复",              │
       │       style: "suspense_twist",             │
       │       duration: 15,                        │
       │       panel_count: 6,                      │
       │       aspect_ratio: "3:4"                  │
       │     }                                      │
       │                                            │
       │                          parse_story 节点   │
       │                          （调用通义千问）      │
       │                               ↓            │
       │                         refine_script 节点  │
       │                          （润色 + 生成       │
       │                           image_prompt）    │
       │                               ↓            │
       │←──── 返回完整脚本 JSON ─────────────────────│
       │     {panels: [{scene, dialogue,            │
       │       camera, emotion, image_prompt}, ...]} │
       │                                            │
       │  3. 展示脚本编辑页（纯前端编辑）               │
       │                                            │
       │  4. 点击「一键生成全部图片」                   │
       │──── POST /api/generate-images (SSE) ──────→│
       │     body: {panels: [...image_prompts]}     │
       │                                            │
       │                     对每格循环：              │
       │                     generate_image          │
       │                     （调用通义万相/Nano）      │
       │                          ↓                 │
       │                     上传 OSS                │
       │                          ↓                 │
       │←──── SSE: {panel_id: 1, image_url: ...} ──│
       │←──── SSE: {panel_id: 2, image_url: ...} ──│
       │←──── SSE: {panel_id: 3, image_url: ...} ──│
       │      ...逐张推送                             │
       │←──── SSE: done ───────────────────────────│
       │                                            │
       │  5. 用户编辑文字（纯前端，无请求）              │
       │                                            │
       │  6. 用户点击某格「重新生成」                   │
       │──── POST /api/regenerate-image ───────────→│
       │     body: {panel_id: 3, image_prompt: ...} │
       │←──── {image_url: "新图片地址"} ──────────────│
       │                                            │
       │  7. 点击「导出」                              │
       │──── POST /api/export ─────────────────────→│
       │     body: {work_id: "xxx"}                 │
       │←──── {images: [...urls],                   │
       │       share_link: "https://..."} ──────────│
       │                                            │
```

# 6. 脚本生成规范

## 6.1 叙事黄金公式

所有脚本必须严格遵循 4 段式叙事结构：

| **阶段** | **名称** | **占比** | **作用** |
| --- | --- | --- | --- |
| 第 1 格 | Hook | 前 20% | 第一秒就抓住注意力，禁止铺垫 |
| 中间 | Escalate | 中间 50% | 快速推进，每格有新信息，情绪递增 |
| 倒数第 2 格 | Twist | 后 20% | 出人意料的反转，打破观众预期 |
| 最后 1 格 | Cliffhanger | 最后 10% | 留钩子，让观众想看下一集 |

## 6.2 禁止项

- 禁止第 1 格用于介绍背景或人物关系
- 禁止连续 2 格情绪强度相同
- 禁止最后一格给完整结局（必须留悬念）
- 禁止任何一格不含对白或旁白
- 禁止连续 2 格相同镜头角度

## 6.3 镜头数与叙事结构映射

| **镜头数** | **Hook (20%)** | **Escalate (50%)** | **Twist (20%)** | **Cliffhanger (10%)** |
| --- | --- | --- | --- | --- |
| 4 | 第 1 格 | 第 2 格 | 第 3 格 | 第 4 格 |
| 6 | 第 1 格 | 第 2-4 格 | 第 5 格 | 第 6 格 |
| 9 | 第 1-2 格 | 第 3-6 格 | 第 7-8 格 | 第 9 格 |
| 12 | 第 1-2 格 | 第 3-9 格 | 第 10-11 格 | 第 12 格 |

## 6.4 8 种脚本风格

| **编号** | **风格** | **英文标识** | **核心特点** |
| --- | --- | --- | --- |
| 1 | 悬疑反转 | suspense_twist | 每格埋伏笔，结尾大反转 |
| 2 | 搞笑沙雕 | comedy_absurd | 夸张表情，无厘头式反转 |
| 3 | 治愈温情 | healing_warm | 温柔细腻，情感反转 |
| 4 | 暗黑爆爽 | dark_revenge | 快节奏强冲突，复仇爽感 |
| 5 | 古风仙侠 | wuxia_fantasy | 古装场景，江湖气息 |
| 6 | 超能力奇幻 | superpower_fantasy | 科幻/魔法元素，觉醒成长 |
| 7 | 恐怖惊悚 | horror_thriller | 压抑氛围，惊吓结尾 |
| 8 | 甜宠恋爱 | sweet_romance | 快节奏对白，反转甜 |

## 6.5 时长选项

时长和镜头数独立，用户自由搭配。

| **时长** | **说明** |
| --- | --- |
| 10秒 | 超短篇，快节奏冲击 |
| 15秒（默认） | 标准漫剧节奏 |
| 30秒 | 中篇，剧情更完整 |
| 60秒 | 长篇，多线叙事 |
| 自定义 | 用户输入秒数 |

## 6.6 图片比例

| **比例** | **说明** |
| --- | --- |
| 3:4 竖版（默认） | 最适合手机漫剧 |
| 1:1 方形 | 适合社交媒体 |
| 16:9 横版 | 适合横屏展示 |
| 自定义 | 用户输入比例 |

## 6.7 缓存规则

相同文案 + 相同参数（风格、时长、镜头数）的脚本结果缓存 7 天，节省 API 调用成本。过期后重新生成。

# 7. 后端服务设计

## 7.1 LangGraph 状态图

### 链路 1：生成脚本（POST /api/generate-script）

parse_story → refine_script → 返回完整分镜脚本

**parse_story：** 接收文案 + 风格 + 时长 + 镜头数，调用通义千问，按黄金公式拆解分镜脚本

**refine_script：** 润色每格描述，补充视觉细节，为每格生成英文 image_prompt

### 链路 2：生成图片（POST /api/generate-images，SSE）

对每一格循环执行：generate_image → 上传 OSS → SSE 推送给 App

**generate_image：** 根据 image_prompt 调用通义万相或 Nano Banana 生图

MVP 不做风格一致性检查（check_style），V2 再加。

### 链路 3：单格操作（直接调用，不经过 LangGraph）

**rewrite-panel：** 直接调用通义千问重写单格脚本

**regenerate-image：** 直接调用生图 API 重新生成单格图片

**generate-single-image：** 为单格首次生成图片
